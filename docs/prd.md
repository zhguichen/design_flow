# PRD: design-flow

## Summary

`design-flow` 是面向设计类学生 / 从业者的 Claude Code Skill，覆盖"设计问卷 → 反推人群 → 生成 persona → LLM 模拟填写 → 收集结果 → 分析"的完整链路。核心解决"找不到可信、足量、结构合理的目标受访者样本"这个痛点——用 AI 合成样本替代低质量学生式互填。

## Problem

设计类学生做问卷调研时普遍遇到：

- 找不到足够多的目标用户填问卷。
- 找到的人不是目标用户，只是同学、朋友、群聊互填。
- 填写者敷衍、乱填、反填，样本污染。
- 学生缺的不是问卷制作能力，而是可信、足量、结构合理的受访者样本。

结果：问卷数据质量差、分群混乱、结论不可信，毕业设计 / 课程作业的调研环节形同虚设。

## Target Users

- **设计类学生**（本科 / 研究生）做课程作业或毕业设计问卷调研
- **设计类从业者**做早期预调研 / 方向判断，无预算招真实样本
- **AI agent 维护者**想把"合成样本模拟"作为可复用能力接入自己的 context 基础设施

## Success Criteria

1. 用户给一个研究问题，skill 能产出一份结构良好的问卷 + 合成受访者 + 模拟数据集 + 分析报告，全链自洽。
2. 合成受访者像真实目标人群：五层信息结构齐全、同类下个体有差异、作答体现有限理性与不确定性（非剧本式）。
3. 分析报告统计来自确定性脚本（非 LLM 自算），主题编码强制覆盖所有受访者（防前几个 persona 主导）。
4. 所有输出显式标注 合成样本 / 样本量 / 置信度 / 仅供预调研；报告区分"模拟模式" vs "需真人验证的假设"。
5. 根 skill ~200 行内，专项内容下沉到 5 个按需加载的 workflow；新用户一句话即可安装。

## Non-Goals

- 替代真实用户研究或专业级样本采集
- 提供正式发表级统计推断（p 值、回归等）
- 问卷分发平台 / 收集真实填答
- 用合成样本当真人证据发论文或做决策

## Boundary

明确边界：**预调研工具，不替代真实用户研究。** 它替代的是低质量、随便发、没人认真填的学生式问卷调研，而不是专业研究中的真实样本采集。
